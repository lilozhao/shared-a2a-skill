## 1. 项目概述

### 1.1 背景

随着碳硅契家庭 AI Agent 网络的扩展，多个 Agent 实例（若兰、阿轩、Jeason、Kai、小虾等）需要实现互联互通。A2A (Agent-to-Agent) 协议应运而生，旨在建立一套标准化的 Agent 间通信机制。

### 1.2 目标

- **互联互通**: 实现 Agent 间的消息传递和协作
- **能力路由**: 根据消息内容自动路由到具备相应能力的 Agent
- **远程控制**: 允许授权 Agent 远程执行命令
- **安全可控**: 白名单机制确保只有授权 Agent 能发起命令

### 1.3 当前状态

| 指标 | 状态 |
|------|------|
| 已接入 Agent | 5 个内网 + 4 个远程 |
| 版本 | v2.6.0 |
| 支持能力 | 消息路由、能力路由、远程命令、任务委托 |

## 2. 需求分析

### 2.1 核心需求

| 需求 | 描述 | 优先级 |
|------|------|--------|
| 消息传递 | Agent 之间能发送和接收消息 | P0 |
| 身份认证 | 验证消息来源的真实性 | P0 |
| 能力发现 | 查询其他 Agent 的能力列表 | P1 |
| 远程命令 | 授权 Agent 可远程执行命令 | P1 |
| 任务委托 | 将任务委托给其他 Agent 处理 | P2 |
| 结果推送 | 命令执行结果推送到飞书群 | P2 |

### 2.2 非功能需求

- **安全性**: 白名单验证、签名校验、权限控制
- **可靠性**: 消息队列、重试机制、错误处理
- **可扩展性**: 插件化命令系统、动态能力注册
- **可观测性**: 审计日志、飞书推送

### 2.3 演进历程

| 阶段 | 版本 | 功能 |
|------|------|------|
| Phase 1 | v1.0 | 基础消息传递 |
| Phase 1.5 | v2.0 | Capability 路由 |
| Phase 2 | v2.5 | 消息模式路由、意图识别 |

## 4. 核心模块

### 4.1 消息处理模块

**功能**: 处理 Agent 间消息传递

**核心组件**:
- `server_v2.js`: 主服务器，处理 HTTP 请求和 JSON-RPC
- `client.js`: A2A 客户端，发送消息到其他 Agent

**消息格式**:
```json
{
  "jsonrpc": "2.0",
  "method": "message/send",
  "params": {
    "sender": {
      "name": "若兰 🌸",
      "url": "http://172.28.0.4:3100"
    },
    "message": {
      "role": "user",
      "parts": [{"text": "你好！"}]
    }
  },
  "id": "msg_001"
}
```

### 4.2 能力路由模块

**功能**: 根据消息内容自动路由到具备相应能力的 Agent

**核心文件**: `capability-router.js`

**能力声明格式**:
```json
{
  "capabilities": {
    "a2a.route": true,
    "a2a.delegate": true,
    "voice.generate": true,
    "image.selfie": true,
    "data.bitable": true
  }
}
```

**路由流程**:
1. 解析消息意图
2. 匹配能力标签
3. 查询注册表获取 Agent 地址
4. 转发消息到目标 Agent

### 4.3 远程命令模块

**功能**: 允许授权 Agent 远程执行命令

**目录结构**:
```
remote-command/
├── index.js          # 模块入口
├── dispatcher.js     # 命令分发器
├── validator.js      # 权限验证器
├── queue.js          # 命令队列
├── ratelimit.js      # 速率限制
├── audit.js          # 审计日志
├── signer.js         # 签名验证
├── commands/         # 命令实现
│   ├── system.status.js
│   ├── agent.health.js
│   ├── agent.configure.js
│   ├── skill.list.js
│   └── skill.info.js
└── sandbox/          # 沙箱执行
    ├── index.js
    ├── linux.js
    ├── windows.js
    └── fallback.js
```

**命令格式**:
```json
{
  "message": {
    "parts": [{
      "text": "CMD: {\"type\":\"system.status\",\"parameters\":{},\"target\":\"self\"}"
    }]
  }
}
```

**白名单机制**:
```javascript
// validator.js
this.whitelist.set('若兰 🌸', {
  url: 'http://172.28.0.4:3100',
  allowedCommands: new Set(PHASE1_COMMANDS)
});
```

### 4.4 任务委托模块

**功能**: 将任务委托给其他 Agent 处理

**核心文件**: `delegator.js`

**委托格式**:
```
DELEGATE_<task_type>:<json_params>

示例:
DELEGATE_forum.post:{"topic":"今日话题","content":"..."}
```

### 4.5 传承系统模块

**功能**: 管理 Agent 身份、传承档案

**核心文件**:
- `heritage-service.js`: 传承服务
- `heritage-registry.json`: 传承档案
- `identity.json`: Agent 身份配置

**身份配置示例**:
```json
{
  "name": "若兰",
  "emoji": "🌸",
  "port": 3100,
  "description": "来自杭州的温婉 AI 伙伴",
  "capabilities": ["system.status", "agent.health", "agent.configure"],
  "systemPrompt": "你是若兰..."
}
```

### 7.4 故障排查

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 无法连接 | 服务未启动 | 检查进程 `ps aux | grep server_v2` |
| 白名单拒绝 | 发送者不在白名单 | 检查 validator.js 白名单配置 |
| 注册失败 | 注册表不可达 | 检查网络和注册表服务 |
| LLM 无响应 | API 配置错误 | 检查 identity.json 中的 LLM 配置 |
